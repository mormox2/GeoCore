# Vision

GeoCore is a knowledge-first foundation for geospatial software.

## Goals

- **Everything is Knowledge** — Model knowledge once; render it as HTML, Markdown, PDF, API, LLM context, mobile, or voice.
- **Interoperability** — Work cleanly with geospatial formats, coordinate systems, and ecosystem tools.
- **Composability** — Small packages combine into larger systems; knowledge objects combine into graphs and collections.
- **Self-hosting** — GeoCore documents itself using its own knowledge model. The project is its first user.
- **Spec-driven implementation** — Codex and contributors implement from [`specs/`](../../specs/), not from scattered notes.

## Non-goals

- Treating FAQs, articles, and guides as separate content types instead of rendered knowledge.
- Polluting the handbook with research dumps or procedural checklists.
- Hiding geospatial complexity behind opaque abstractions.

## Success criteria

GeoCore succeeds when:

- Knowledge objects are stable, spec-defined, and renderer-agnostic.
- The [`knowledge/`](../../knowledge/) tree is a living instance of the model.
- Teams adopt packages incrementally without relearning the entire system.
