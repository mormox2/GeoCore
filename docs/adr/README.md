# Architecture Decision Records

ADRs capture **architecture decisions only**.

For product, UX, AI, or GEO decisions, use [`docs/decisions/`](../decisions/).

## Architecture includes

- System structure and package boundaries
- Technology and integration choices
- Data flow and renderer architecture
- Security and infrastructure patterns

## Filename format

`NNNN-short-title.md`

## Template

```markdown
# Title

- Status: proposed | accepted | deprecated | superseded
- Date: YYYY-MM-DD

## Context

## Decision

## Consequences
```
